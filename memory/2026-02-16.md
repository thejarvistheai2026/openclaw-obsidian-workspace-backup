## Website Deployment Saga (12:57 - 18:30 EST)

### What Happened
Spent ~6 hours debugging why Franco's personal website wouldn't deploy to DigitalOcean. Build succeeded, server started, but every route returned 404.

### Root Cause
DigitalOcean App Platform was in a broken state where the compiled `.next` folder wasn't being included in the runtime container. The app would build successfully but the server couldn't find any routes at runtime.

### What We Tried (and failed)
1. ✅ Checked build logs - all green
2. ✅ Changed `next.config.js` to `output: 'standalone'`
3. ✅ Modified `package.json` start script to use `server.js`
4. ✅ Added `postbuild` to copy `content/` folder
5. ✅ Added debug logging to trace filesystem paths
6. ✅ Created `/api/debug` endpoint to inspect runtime
7. ✅ Removed standalone mode, went back to standard Next.js
8. ✅ Created `.do/app.yaml` config file

### What Actually Worked
**Destroyed the DO app and created a new one.** Fresh app from same GitHub repo deployed perfectly on first try. All routes working.

### Key Lesson
> **When DO App Platform acts weird, nuke it and rebuild.** Don't waste hours debugging mysterious 404s when the code is correct. The platform can get into broken states.
