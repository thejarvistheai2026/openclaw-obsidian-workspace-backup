## Deployment Fix: TypeScript Narrowing (20:47 EST)

### Problem
DigitalOcean build failed with:
```
Type error: Property 'metadata' does not exist on type 'never'.
./app/projects/[slug]/page.tsx:42:18
```

### Root Cause
`notFound()` from Next.js returns `never`, but without an explicit `return` statement, TypeScript doesn't properly narrow the variable type after the guard clause.

### Files Fixed
1. `app/projects/[slug]/page.tsx` - added `return;` after `notFound()`
2. `app/projects/tag/[tag]/page.tsx` - added `return;` after `notFound()`
3. `lib/content.ts` - kept placeholder (returns null for projects)

### Pattern for Future
```typescript
const data = await fetchData();

if (!data) {
  notFound();
  return;  // <-- Always add this for TypeScript narrowing
}

// data is now properly typed, not `never`
```
