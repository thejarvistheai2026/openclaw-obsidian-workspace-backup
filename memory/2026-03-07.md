## Update: Discord Backfill Added

**13:35 EST** - Created historical message backfill capability:

**Why:** Real-time capture only gets messages from when bot.js starts. Historical data valuable for:
- Full retro context
- Pattern analysis  
- Training data

**Files Created:**
- `discord-capture/backfill-channel.js` - Fetches message history
- `discord-capture/backfill.sh` - Easy-to-use wrapper script

**Usage:**
```bash
cd discord-capture
./backfill.sh        # Backfill 30 days (default)
./backfill.sh 7      # Backfill 7 days
./backfill.sh 90     # Backfill 90 days
```

**How It Works:**
- Fetches from all accessible text channels
- Respects Discord rate limits (1s delay between batches)
- Auto-sorts by original date into `raw/YYYY-MM-DD.jsonl`
- Marks as `"type": "historical"` to distinguish from real-time
- Can be run multiple times (uses message ID, won't duplicate)

**Tested:** Bot now captures from 12+ channels in real-time:
- general, links, remix
- project-newsletter, project-oc-setup-guide
- project-appaudits-site, project-fox-valley-tracker
- project-francosite, project-gadget
- project-notion-md, project-transcripts
- project-voice
