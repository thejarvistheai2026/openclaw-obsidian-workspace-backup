## Critical: Gateway Restart Pattern

**Issue Discovered:** Using `openclaw gateway stop/start` breaks service

**What Happened:**
- Debugging Discord required gateway restart
- Used `stop` then `start` instead of `restart`
- Result: LaunchAgent unloaded, service not auto-starting

**Why:**
- `stop` = unregisters from macOS LaunchAgents
- `start` = runs manually in foreground
- After `stop`/`start`, service won't auto-restart on crash or boot

**The Fix:**
```bash
# After accidentally using stop/start:
openclaw gateway install   # Re-registers LaunchAgent
openclaw gateway restart

# For normal config changes, ALWAYS use:
openclaw gateway restart   # ✅ Keeps LaunchAgent registered
```

## Discord Intent Lessons

**What We Learned:**

1. **Discord REST API ≠ Gateway WebSocket**
   - API shows intents enabled
   - Gateway can still report "limited"
   - They sync eventually, but can lag

2. **Token Regeneration ≠ Intent Fix**
   - New token forces re-auth
   - But if portal had issues, still shows "limited"
   - Requires: portal toggle OFF → Save → ON → Save → new token → restart

3. **Permission 0 Issue**
   - When re-inviting, if OAuth URL has `permissions=0`, bot has no perms
   - Must use proper OAuth URL generator with permissions included
   - Required perms: View Channels, Send Messages, Read Message History

**Next Action:** Generate proper OAuth URL with permissions and re-re-invite bot.

- OpenClaw 2026.2.14 rejected it as "Unrecognized key: messageContent"
- Gateway crashed, required manual recovery via ChatGPT + file editing
- Franco had to manually remove invalid keys

**Root Cause:**
- Docs referenced `intents` configuration (likely for newer version 2026.3.7+)
- Version 2026.2.14 doesn't support this config key
- Should have checked `openclaw config schema` first

**Safety Rule Added:**
- NEVER add unknown config keys without validating against schema first
- Always use: `openclaw config schema | grep <key>` to verify supported keys
- Before editing sensitive configs, run `openclaw doctor` first
- Back up config before any changes

**Discord Issue Root Cause Identified:**
- OpenClaw 2.14's Discord client cannot properly request Message Content Intent
- Portal toggle is ON, but WebSocket gateway handshake still reports "limited"
- REST API works (can read/send messages), but WebSocket events don't fire
- This is a version compatibility issue between OpenClaw 2.14 and current Discord gateway

**Resolution Path:**
- Option A: Update to OpenClaw 2026.3.7 (risky, requires update plan)
- Option B: Use alternative access method (polling-based Discord capture)
